\hypertarget{README.md_autotoc_md142}{}\doxysubsubsection{\texorpdfstring{{\ttfamily auto\+Label}}{{\ttfamily auto\+Label}}}\label{README.md_autotoc_md142}
`\textquotesingle{}dev-\/only' \texorpdfstring{$\vert$}{|} \textquotesingle{}always\textquotesingle{} \texorpdfstring{$\vert$}{|} \textquotesingle{}never\textquotesingle{}{\ttfamily , defaults to}dev-\/only\`{}.

This option enables the following\+:


\begin{DoxyItemize}
\item Automatically adds the {\ttfamily label} property to styles so that class names generated by {\ttfamily css} or {\ttfamily styled} include the name of the variable the result is assigned to.
\item Please note that non word characters in the variable will be removed (Eg. {\ttfamily icon\+Styles\$1} will become {\ttfamily icon\+Styles1}) because {\ttfamily \$} is not valid \href{https://stackoverflow.com/questions/448981/which-characters-are-valid-in-css-class-names-selectors\#449000}{\texttt{ CSS Class\+Name Selector}}
\end{DoxyItemize}

Each possible value for this option produces different output code\+:


\begin{DoxyItemize}
\item with {\ttfamily dev-\/only} we optimize the production code, so there are no labels added there, but at the same time we keep labels for development environments,
\item with {\ttfamily always} we always add labels when possible,
\item with {\ttfamily never} we disable this entirely and no labels are added.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md143}{}\doxysubsubsubsection{\texorpdfstring{css}{css}}\label{README.md_autotoc_md143}
