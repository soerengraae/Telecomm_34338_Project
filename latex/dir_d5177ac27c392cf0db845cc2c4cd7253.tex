\hypertarget{README.md_autotoc_md3339}{}\doxysubsubsubsection{\texorpdfstring{delaunay.\+halfedges}{delaunay.\+halfedges}}\label{README.md_autotoc_md3339}
A {\ttfamily Int32\+Array} array of triangle half-\/edge indices that allows you to traverse the triangulation. {\ttfamily i}-\/th half-\/edge in the array corresponds to vertex {\ttfamily triangles\mbox{[}i\mbox{]}} the half-\/edge is coming from. {\ttfamily halfedges\mbox{[}i\mbox{]}} is the index of a twin half-\/edge in an adjacent triangle (or {\ttfamily -\/1} for outer half-\/edges on the convex hull).

The flat array-\/based data structures might be counterintuitive, but they\textquotesingle{}re one of the key reasons this library is fast.\hypertarget{README.md_autotoc_md3340}{}\doxysubsubsubsection{\texorpdfstring{delaunay.\+hull}{delaunay.\+hull}}\label{README.md_autotoc_md3340}

An array of input coordinates in the form {\ttfamily \mbox{[}x0, y0, x1, y1, ....\mbox{]}}, of the type provided in the constructor (or {\ttfamily Float64\+Array} if you used {\ttfamily Delaunator.\+from}).\hypertarget{README.md_autotoc_md3342}{}\doxysubsubsubsection{\texorpdfstring{delaunay.\+update()}{delaunay.\+update()}}\label{README.md_autotoc_md3342}
